JAVA学习个人笔记

仅仅记录个人学习java时学到的知识和心得

类

由类定义一个具体的对象的组织和其上的操作。 当声明一个具体的存在对象时，类在其上的定义可以帮助辨识这个存在的组织结构和操作。

对象分为实际的类别和引用的类别。

JAVA中的变量实际上都是引用，类似于C中的指针为指向性信息。
不同的是，这些指向性信息的类别不一定和指向的对象的真实类别相符合，这也就实现JAVA的一个特性：多态。

字段(field)

对应于方法的属性，表明了一个类相关方面的属性。

在class中定义的字段称为实例字段，其特点是每个实例都有独立的字段且占用不同的空间，互不影响。
但是由static修饰的字段为静态字段，只占有一个”共享“空间，且所有实例共享该字段。
更改一个实例的静态字段会改变与之相同类的所有实例的字段值，该静态字段实际上指向的都是类的静态字段，因此在访问的时候尽量使用类名.静态字段

接口

抽象方法本质上是定义了接口(interface)规范，接口可以说，比如一个对象是一个具体类的实例(instance)，那么其方法可以是这个对象或者说这个类的接口，提供了类(对象)与外界(系统，用户)的通信，接口类即只提供了抽象方法的类，只说明了有哪些通信的方法，这为子类的继承提供了规范。
可以理解为是相关事物的全集，比如对所有人类来说的全集是人类这个物种(从一个角度进行说明)，则人类这个类就是一个接口，声明了人类有那些行为，具体的需要人单独的个体(对象，instance)进行说明。

接口类(interface)不能有字段，当用interface声明一个接口(interface name)时，不能有字段，同时所有的方法默认为public abstract。

此时子类不再用extends而是implements，extends只能继承一个类，而implements可以是实现同时继承多个interface(用’,’隔开)。

JAVA的接口(interface)特指其定义，表示一个接口类型和一组方法签名。

接口可以继承(extends)自另一个接口。

抽象程度

接口(interface)>抽象类(abstract)>实例(具体指明的类)(instance)

接口的静态方法

接口可以定义静态字段，此时静态字段得要final类型进行修饰，因为只能是这样，因此实际上interface可以定义字段，不过会自动使用public static final修饰

default方法

接口可以定义default方法(类似于lambda)，即接口定义一个拥有具体执行步骤的方法(类似于所有的子类和示例都具有完全一致的方法，那么可以直接在接口中进行定义，比如全体人类都使用肺进行呼吸)。
default方法的目的是，当我们需要给接口新增一个方法时，会涉及到修改全部子类。如果新增的是default方法，那么子类就不必全部修改，只需要在需要覆写的地方去覆写新增方法。

静态方法

与静态字段相似的方法，使用类名就可以进行调用。

静态方法通常用于工具类和辅助方法。

this

this表示类当前被调用的实例

*静态方法无法访问实例字段，只能访问静态字段（static）

包(package)

用于管理类，当它人要使用类和接口时，使用包能更好地进行区分并且不会造成类名相同导致的混乱。

在每一个类文件前加上package [包名]，表示当前文件中的类只能被包名.类名调用。

包无父子关系，但所有java文件的目录层次要和包层次相同。以’.’作为分割。

位于同一个包的类，可以访问包作用域的字段和方法。不用public、protected、private修饰的字段和方法就是包作用域。

bin目录主要存放的是已经编译好的java文件以便调用，可以使用javac -d bin目录 文件1 文件2……文件n 自动进行编译。

使用import进行引用，在编写class时，编译器(JAVAC)会自动做两个动作：

  1. 默认自动import当前package的其他class
  2. 默认自动import java.lang.*

作用域

即可以被访问的范围。

public的class和interface可以被其他任何类访问（需要import）。

public的field和method可以被访问但是需要有访问对应class的权限。

private的field和method无法被其他类访问，在进行编写的时候尽量把public方法写在前面。

Java支持嵌套类，那么嵌套类可以访问private。

包作用域(package)指一个类可以访问同一个包中没有被public，private修饰的class以及没有public，protected，private修饰的字段和方法。

局部变量指的是在方法内部定义的变量，其作用域从变量声明处到对应的块（{}）结束处。

final修饰符(相当于一个常量)可以：

  1. 阻止class被继承。
  2. 阻止子类覆写class的方法。
  3. 阻止field被重新赋值。
  4. 阻止局部变量被重新赋值。

一个.java文件只能包含一个public类，但可以包含多个非public类。

内部类

定义在class内部的class称为嵌套类(nested class)。

在实例化的时候需要类.new 嵌套类()

匿名类

因为通常内部类不关心类名，只需去调用，因此使用匿名类。匿名类也可以继承自普通类。

使用static修饰的内部类，不依附于其他类而是一个独立的类，仍然可以访问外部类的private字段和静态方法。

即内部类分为三种：

  1. 内部普通类
  2. 匿名类
  3. 静态内部类

ClassPath

classpath是JVM(Java Virture Machine)使用的环境变量，指示JVM如何搜索class(javac编译成字节码文件，JVM负责运行)

JVM用于提供环境运行class文件

因此classpath是一组目录的集合，设置的搜索路径与操作系统有关

Windows

使用;分割

Linux

使用:分割

在设定的目录下找对应的class文件。

classpath的设定：

  1. 在系统环境变量中进行设定。**(不推荐)**
  2. 在启动JVM的时候进行设定。**(推荐)**

使用java命令时即启动JVM运行对应的文件，使用-classpath参数或者-cp。

若没有设定，默认为当前目录。若在IDE中，则自动传入当前工程的bin目录和引入的jar包。

jar包

比如有多个class文件，散落于各级目录，若每次调用都使用参数一一填入会相当麻烦，因此引入jar包，可以把package组织的目录层级以及各个目录下的文件打包成一个jar文件。

jar包实际是一个zip格式的压缩文件，相当于目录，因此我们可以把项目打包成jar包，这样我们可以使用JVM运行的时候指定classpath为jar包，方便调用其中的类和其他事物(jar包记录了package的组织层次，相当于一次性制定了classpath的目录)

jar的制作，找到正确的目录将其打包成zip文件并更改后缀名便可。jar包的是第一层目录应该为bin下(存放class文件的目录)各个package的第一层目录。

由于jar文件本身是多个class文件的集合，因此在运行的时候需要指定一个入口class(相当于整个程序的入口)，这是我们可以的使用/META-INF/MANIFEST.MF文件指定Main-Class和其他信息，放至jar包中，JVM会自动进行读取。

此时可以使用java -jar name.jar运行。

实际上为java -cp name.jar Main-Class。

jar包中可以包含其他jar包，需要

在MANIFEST.MF中配置classpath，Java社区提供的开源构建工具Maven可以方便地创建jar包。

rt.jar

JVM自带的Java标准库也是以Jar文件的形式进行存放，即rt.jar

因此在java9之前，自己编写的jar文件中会依赖很多其他的库(jar)文件，导致运行时要一一进行指明。

模块

为了解决依赖这个问题，从Java9开始引入了模块，让class容器能自带”依赖关系“，使得编译运行时能自动定位并解决依赖关系。

因此rt.jar被拆分成几十个模块并以.jmod拓展名标识。模块名即是文件名。

在jvm标准库中的模块文件中，模块之间的依赖关系已经由module-info.class文件写入，所有的模块之间或者间接依赖java.base模块。

使用方法

首先建立工程文件，在src源码下构建module-info.java文件(文件名只能是此)，之后在其中编写模块依赖。

进行编译，使用代码在当前目录中使用

javac -d bin /src/module-info.java 已有的源码

打包成jar，使用

jar –create –file name.jar –main-class 指定入口类 -C bin

即在当前目录得到对应的jar包

*jar包可以设置classpath，运行的时候自动加载

若要转换成模块，则我们使用

jmod create –class-path name.jar name.jmod

得到我们自己定义的模块

打包jre，即得到我们自己的java运行环境，因为在实际的应用中，我们并非完全需要jvm中所有的标准库文件，因此我们可以通过使用自己的模块来构建一个jre环境，换句话来说，得到了一个可以精简版的jre环境，且这个环境可以给别人使用并且给我们的程序“瘦身”

访问权限

若一个模块要访问另一个模块中的类，则被访问的模块的模块解释文件中要使用

exports 对应的完整类名

这样才能进行访问。

Unicode和UTF-8

Unicode编码需要两个或者更多字节表示，而UTF-8是变长编码，依靠高字节确定一个字符究竟是几个字节，因此在数据传输中具有较好的容错性。

Java中的char类型就是两个字节的Unicode编码。

Java中，我们可以直接使用+拼接字符串，但是由于字符串的不可改变的性质，每次拼接后得到的实际上是复制原先的对象，而原先的对象若不进行使用就会被舍弃，会浪费内存还有GC效率。

GC:即找到不再使用的对象，回收对象并释放对应的内存。

因此为克服这种情况带来的影响，使用StringBuilder类（可变字符串对象），此时往对象中添加字符不会创建新的临时对象。

因为此类使用的是return this，也就是每次拼接后返回的仍然是原对象，避免了多余对象的产生。

静态工厂方法

Integer.valueOf()是Integer包装类的一个方法，在Java中进行一定的优化，比如对较小的数不会重新再构建一个实例而是返回值相同的已有实例，能节省时间和空间（内部缓存优化），因此在创建Integer实例时，我们尽量可以使用Interger.valueOf()进行创建，这样有助于节省内存等。

因此能够创建新对象的静态方法即为”静态工厂方法“，会尽可能地返回缓存中的实例来节省内存。

因此我们在创建对象时优先选用静态工厂方法。

Java异常处理

Java使用异常表示错误

异常是一种class，表示类型信息。

可以使用try...catch语句来捕获异常，即当抛出异常时执行对应的语句。不捕获往往会造成无法编译等问题。

在定义方法时可以使用throw Xxx表示该方法可能抛出的异常类型，非RuntimeException调用时必须强制捕获这些异常，RuntimeException类无需。

在使用多try...catch语句时，要把子类写至前面，因为类似if，假如抛出的是子类的异常，那么父类写前面一定会报父类的错误而不是子类。即根据子类关系排顺序。

finally语句，在try...catch语句中，此语句总是最后执行，保证有无错误都会在最后执行这个语句。

我们可以使用“|”符号来同时捕获多个异常，相当于或逻辑，多用于多个异常的处理方式相同时。

异常体系包含两大类错误Error和Exception，error表示严重的错误，exception表示运行时产生的错误，可以被捕获并进行处理。

Java标准库定义的常用异常

自定义异常时常常先自定义一个根异常，此根异常常常先从原有的异常中进行派生，例如extends RuntimeException，之后以此为基础进行自定义异常的构建。同时根异常应该提供多个构造方法（可参考标准库给出的异常的构建方法）。

当出现NullPointerException异常时，在Java14以上，可以给JVM添加一个-XX:+ShowCodeDetailsInExceptionMessages参数，使得在出现此异常时，能精确指出是哪个字段等导致了这个异常。

分为两大类

  • RuntimeException及其子类，无需强制捕获
  • 非RuntimeException，即Checked Exception，需要强制捕获，即使不在调用层进行捕获也需要在更高的调用层进行捕获，main()方法是最后捕获异常的机会。使用throw声明了异常需要捕获否则无法正常报错。

Java日志

在实际项目的调试过程中，我们常常需要对出错的地方进行查看，此时可以通过日志进行错误的记录。

Java提供了一个第三方库Commons Logging，是由Apache创建的日志模块，其特点是可以挂载不同的日志系统，默认挂载使用Log4j，若没找到则使用JDK Logging（Java自带的日志系统），可以理解为一个日志接口，而日志的实现（实例）为Log4j或其他日志系统。这样的好处是可以在开发和实现阶段无需修改代码（代码使用接口），而修改使用的日志系统的配置即可方便日志的使用。

覆写equals方法

推荐写法

equal方法要求

  1. 自反性，x.equals(x)应为真
  2. 对称性，x.equals(y) 和 y.equals(x) 同真假
  3. 传递性，x.equals(y) 真，y.equals(z) 真， 则x.equals(z)为真
  4. 一致性，反复调用相同euqals式应返回同一真假
  5. x.equals(null) 应为假

equals方法建议

  1. 显式参数命名为otherObject，稍后转换成other
  2. 检测this与otherObject是否引用同一个对象
  3. 检测otherObject是否为null
  4. 比较是否属于同一类 - 子类能拥有自己的相等概念，使用getClass();由超类决定相等，使用instanceof（）

避免破坏封装性

“hello”字面值常量直接在公共池内创建对象，new会在对应的类池内构建对象

但注意不要让两个引用变量引用同一个对象，尤其是一个引用变量是一个实例的内部成员变量。

即使String类并没有直接修改其内存中对象的方法，但是这么做会破坏封装性。

对象克隆

对象克隆分为两种：

  1. 浅拷贝
  2. 深拷贝

浅拷贝，对象本身拷贝新一份（大小相同的对象），但是其内部域不会进行拷贝，比如内部成员变量在原变量和克隆变量中引用的仍是同一个对象。

因此Object中的clone方法是protected，默认的克隆方法是浅拷贝

要使用clone方法，需要类：

  1. 实现cloneable接口规范（implements）
  2. 使用public修饰符覆写clone方法

*所有的数组类型均包含一个public的clone方法

java启动参数

java启动程序的方式：

  1. java [ options ] SomeClass [ arguments ]
  2. java [ options ] -jar SomeJar.jar [ arguments ]

options指定java虚拟机运行的一些参数，称为系统参数
arguments为运行参数，传入程序内部的值

(jar启动的jar包内需要指定入口类)

虚拟机参数（系统参数）类别：

  1. 标准参数（-）：所有的JVM实现都必须实现这些参数的功能，而且向后兼容。
  2. 非标准参数（-X）：默认JVM实现这些参数的功能，但是并不保证所有JVM实现都满足，且不保证向后兼容。
  3. 非Stable参数（-XX）：此类参数各个JVM实现会有所不同，将来可能会随时取消，需要慎重使用

java标准参数

  • -classpath 设置class文件目录

比如类文件在l目录下的com.tmp包内，cp指定路径为l，那么需要com.tmp.文件名进行调用

系统属性参数
系统属性参数以-D<propertyName> = value形式定义，运行在此虚拟机之上的应用程序可用System.getProperty("propertyName")得到value的值。

maven系统属性参数

  • -Dmaven.multiModuleProjectDirectroy maven构建的项目目录（pom.xml根目录）
  • -Dmaven.home maven程序目录
  • -Dclassworlds.conf 设置maven所需的jar包
  • -Dfile.encoding 指定文件编码格式
  • -Dexec.workingdir 设置程序运行的根目录
  • -Dexec.args 设置程序运行的命令

非标准参数

非标准参数又称为扩展参数，是指以-X开头的参数，可以通过java -X打印出所有的非标准参数。
常见的非标准参数如：

  1. -Xms：指定JVM堆的初始大小（memory of start?），如-Xms3550m。
  2. -Xmx：指定JVM堆的最大值（memory of maximum?），如-Xmx3550m。
  3. -Xmn：指定JVM堆中新生代的大小（memory of nursery），如-Xmn2g。
  4. -Xss：设置单个线程栈的大小（stack size），一般默认为512k。等同于-XX:NewSize。
  5. -Xprof：跟踪正运行的程序，并将跟踪数据在标准输出输出；适合于开发环境调试。
  6. -Xloggc:：与-verbose:gc功能类似，只是将每次GC事件的相关情况记录到一个文件中。若与verbose命令同时出现在命令行中，则以-Xloggc为准。
  7. -Xint：设置JVM以解释模式运行，所有的字节码将被直接执行，而不会编译成本地码。

字节输入流（抽象父类）</li> <li>java.io.OuputStream 字节输出流（抽象父类）</li> <li>java.io.Reader 字符输入流（抽象父类）</li> <li>java.io.Writer 字符输出流（抽象父类）</li> </ul> <p>按流向可分为：输入流和输出流<br>按输入输出的位置可分为：文件、进程或线程（管道流）、其他流（过滤流）<br>按输入输出的对象可分为：字节、字符、具体对象（逻辑类型数据）<br>按输入输出的阻塞可分为：非阻塞（面向缓存的buffer oriented）、阻塞</p> <h3 id="字节流"><a href="#字节流" class="headerlink" <p>ByteArrayOutputStream类：字节数组输出流，向内存中存储的字节数组输出</p> <p>BufferedInputStream类：利用缓冲区读取数据，可以一次性读取多字节的数据</p> <p>BufferedOutputStream类：利用缓冲区存储输入流，使用flush()来将缓冲区的内容输出到外存</p> <p>PipedInputStream类：管道流，用于进程和线程之间的数据交换</p> <p>PipedOutputStream类：可以使用connect()与管道流建立连接，Output -> Input</p> <p>DataInputStream类：过滤流，其父类为BufferedInputStream</p> <p>PrintStream类：其创建的输出流能输出格式化的数据</p> <p><strong>System类</strong></p> <p>可以通过setIn(),setOut(),setErr()来将输入输出重定向</p> <h3 id="字符流"><a href="#字符流" class="headerlink" title="字符流"></a>字符流</h3><p>java.io.Reader / java.io.Writer 字符流抽象类</p>
